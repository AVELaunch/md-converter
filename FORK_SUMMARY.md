# Fork Summary: AVELaunch/md-converter vs. Gadamad/md-converter

This fork extends the original md-converter with local OCR support,
security hardening, a proper macOS app bundle, and expanded tests.
Primary motivation: run it locally as a private tool with no cloud
dependencies and no IP/content-policy refusals (books, technical PDFs,
scanned materials).

## Headline changes

- **Marker OCR as primary engine** — scanned PDFs now go through a three-tier pipeline (PyMuPDF text → Marker deep-learning OCR → Tesseract fallback), all local, no cloud, no content filters.
- **SSRF hardening on URL fetch** — `_safe_get()` blocks private IPs (including redirects and the final landed URL), rejects non-HTTP schemes, caps responses at 50 MB.
- **27 tests** up from 3, covering every converter, SSRF guards, YAML escaping, OCR engine dispatch, `route()`, and `convert_pasted()`.
- **Standalone macOS .app bundle** with proper `Info.plist` identity, custom `.MD↓` icon, and NSBundle override so the Dock shows "MD Converter" instead of "Python".
- **Removed legacy scripts** — `pdf_to_md.py` (which contained `os.system("pip install ...")` on ImportError), `pdf_ocr_to_md.py`, and `docx_to_md.py`, all superseded by `src/converters.py`.

## Detailed changes by area

### Security

- `_safe_get()` replaces raw `requests.get()` for all URL fetches. Resolves hostnames via `socket.getaddrinfo` and rejects any IP where `is_private`, `is_loopback`, `is_link_local`, `is_reserved`, or `is_multicast` is true — blocking SSRF to `169.254.169.254`, `127.0.0.1`, RFC 1918 ranges, etc.
- Every redirect hop in `resp.history` and the final `resp.url` are validated against the same private-IP check.
- `file://` and other non-HTTP schemes are rejected before any network call.
- Response bodies are streamed with a 50 MB cap (`_MAX_RESPONSE_BYTES`).
- YAML frontmatter values (`title`, `source`) are escaped via `json.dumps()` when they contain quotes, backslashes, newlines, or YAML-unsafe leading characters. Round-trip verified with `yaml.safe_load` in tests.
- The module-level `os.environ["SSL_CERT_FILE"] = _CA_FILE` mutation was scoped to run only inside `_safe_get()`, not at import time.
- Deleted `pdf_to_md.py` which ran `os.system("pip install pymupdf")` on ImportError — arbitrary shell execution triggered by a missing dependency.

### OCR pipeline

- Three-tier engine: PyMuPDF text extraction (fastest, for text-layer PDFs) → Marker (`marker-pdf`, deep-learning OCR) → Tesseract (`pytesseract`, traditional OCR fallback).
- Engine selection controlled by `MD_CONVERTER_OCR` env var or `ocr_engine` key in `config.json`. Values: `auto` (default — Marker first, Tesseract on failure), `marker`, `tesseract`.
- `PdfConverter` instance is cached at module level (`_MARKER_CONVERTER`) so batch OCR of multiple scanned PDFs doesn't reload ~1 GB of models per file.
- "Loading Marker models" is logged once per process at INFO; subsequent calls log at DEBUG.
- Marker model weights (~1 GB) lazy-download to `~/.cache/` on first OCR run. Optional prefetch via `MD_CONVERTER_PREFETCH=1 bash install.sh`.

### Code quality

- Structured logging via `logging.getLogger("md_converter")` across `converter_app.py`, `converters.py`, and `native_drop.py`. All `print("[native_drop] ...")` diagnostics replaced with `logger.warning()`/`.error()`.
- Every bare `except: pass` and `except Exception: pass` replaced with specific exception types (`RuntimeError`, `OSError`, `subprocess.SubprocessError`) and logged with context.
- `logging.basicConfig()` configured in `main()` — INFO level to stderr.
- 27 tests (up from 3 in upstream's `test_cli_mode.py`): round-trip tests for all 6 format converters, SSRF rejection (file scheme, loopback, metadata IP), size cap, YAML frontmatter escaping round-trip, frozen-build path resolution, OCR engine dispatch (Tesseract-only, auto-fallback), `route()` dispatch for each extension and unknown types, `convert_pasted()` for plain text, URL routing, and empty input.
- Deterministic test fixtures generated by `tests/_make_fixtures.py`: `.txt`, `.html`, `.docx`, `.pdf` (text), `.pdf` (scanned/image-only), `.rtf`.
- Pipe characters in DOCX table cells are escaped to `\|` so they don't break Markdown table syntax.
- Conventional Commits throughout.

### Architecture

- `src/paths.py` is the single source of truth for `user_data_dir()` and `config_path()`. Both `converter_app.py` and `converters.py` import from it — no duplicated path logic.
- Frozen-build detection via `getattr(sys, "frozen", False)`. When frozen, outputs go to `~/Library/Application Support/MD Converter/converted/` and config reads from the same directory. When running from source, paths resolve relative to the repo root (unchanged from upstream).
- `FetchResult` named tuple cleanly separates raw bytes and encoding from the HTTP response, replacing the previous direct `resp.text` usage.
- `_get_ocr_engine()` uses `_config_path()` from `paths.py`, so it finds the correct config in both frozen and dev modes.
- PyYAML added as a dependency (used for frontmatter validation in tests).

### macOS packaging

- `scripts/launch.command` — double-clickable launcher with portable path resolution (`dirname "$0"/..`), explicit `PATH` export for Homebrew Python, and `exec` for clean process replacement.
- Standalone `.app` bundle at `/Applications/MD Converter.app` with `Info.plist` (CFBundleName, CFBundleIdentifier `co.avelaunch.mdconverter`, CFBundleIconFile), ad-hoc codesigned.
- PyObjC `NSBundle.mainBundle()` override in `converter_app.py` sets `CFBundleName` and `CFBundleDisplayName` at runtime so the Dock shows "MD Converter" instead of the Python interpreter's name.
- Custom `.MD↓` app icon: source iconset (10 PNGs), compiled `.icns`, and `scripts/generate_icon.py` for regeneration. All committed under `assets/`.
- `install.sh` updated with optional Marker model prefetch step.

### Legacy cleanup

- Removed `pdf_to_md.py`, `pdf_ocr_to_md.py`, `docx_to_md.py` from repo root — all superseded by `src/converters.py` unified dispatch via `route()`.
- The old `scripts/generate_icon.py` (Pillow-based icon with "MD" text and arrow on dark rounded rect) replaced with the new `.MD↓` design.

## Known limitations carried forward

- Frozen PyInstaller `.app` builds from `scripts/build_app.sh` do **not** yet bundle Marker OCR dependencies (`marker-pdf`, `torch`, `transformers`, `surya-ocr`), certifi CA bundle, or torch data files. Scanned-PDF conversion in frozen builds falls back to Tesseract. Documented as a known issue in `CHANGELOG.md` and a TODO in `build_app.sh`.
- DNS-to-request TOCTOU in `_safe_get()`: hostname is resolved and validated, then `requests.get()` resolves it again independently. A DNS rebinding attack could theoretically exploit the gap. Low risk for a local tool.
- `safe_filename()` can produce collisions for titles that differ only in special characters — no deduplication logic. Second file silently overwrites the first.
- No per-page progress feedback during long OCR runs in the GUI; progress bar jumps from 0% to 100% per file.

## What's unchanged

- Core GUI structure: single-window pywebview app with inline HTML/CSS/JS, drag-and-drop zone, URL input, log panel, progress bar.
- `native_drop.py` ObjC swizzle approach for real Finder file paths (logging changed from `print` to `logger`, but mechanism identical).
- Obsidian vault integration: optional `config.json` with `vault_path`, YAML frontmatter with `tags: [converted, <type>]`, vault delivery organized by source type.
- Output format: `# Title` header, blockquote metadata (source, word count, extras), `---` separator, body. Unchanged.
- `config.json` format: still optional, still gitignored, `vault_path` key unchanged. Only addition is the new `ocr_engine` key.
- All six format converters (PDF, DOCX, HTML/URL, TXT, RTF, pasted text) — same conversion logic, same output structure.
- `striprtf` latin-1 decode strategy for RTF files.
- CLI mode (`sys.argv > 1` dispatch, tabular summary output).
